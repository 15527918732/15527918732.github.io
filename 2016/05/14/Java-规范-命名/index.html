Java 规范-命名

  • Java 重构 设计

1.Package的命名规范 （全部小写，由域名定义）

cn.edu.usst.dingyuewei.package

2.Class命名规范 -（单词首字母大写） 

当要区别接口和实现类的时候，可以在类的后面加上“Impl”-interface IContainer class ContainerImpl

Exception类最好能用“Exception”做为类命名的结尾 DataNotFoundException InvalidArgumentException

抽象类最好能用“Abstract”做为类命名的开头 AbstractBeanDefinition AbstractBeanFactory

Test类最好能用“Test”做为类命名的结尾 ContainerTest`

3.Interface 能力附加型+able 

Runable

4.方法命名:动名词-（首字母小写，字母开头大写）

不同动作代表不同含义

create：创建

get/set:获取或设置

to：转换

is/exist:代表返回不同

4.变量命名

具体含义+反应变量类型前缀或后缀

String-str，Array-atrr，Buffer-buf

循环计数变量通常采用字母 i，j，k

5.常量命名-（全部大写 ，常加下划线）

public static final String NUM_DAYS_IN_WEEK="123";

6.注释规范

方法默认Javadoc（/*快捷键）

复杂逻辑 添加注释
