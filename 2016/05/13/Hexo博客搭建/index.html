Hexo博客搭建

  • 开发,框架

其实先前一直就有这个打算，就是因为一些琐事，迟迟没有写，今天终于下定决心，搭建这个HEXO博客环境，希望能把自己经历的一些点点滴滴记录下来，与大家一起分享。 HEXO，是一个比较成熟的静态博客框架直接就被我借鉴过来，搭建了我第一个博客环境.俗话说，工欲善其事，必先利其器。做开发的一样，好的工具，好的软件，会让我们的开发工具会让我们做起事来事半功倍。

1. 配置环境

NodeJs:用来生成静态页面的，hexo就是基于nodejs的，到Node.js官网下载相应平台的最新版本，一路安装即可。

Git:把本地的hexo内容提交到github上去

GitHub:服务需要发布网上，大家才能访问，github支持静态资源访问，所以必须要有github账号

2.安装

步骤:

1.sudo npm install-g hexo hexo基本配置文件

2.hexo init hexo初始化

3.hexo generate 生成静态页面

4.hexo server 启动本地服务 浏览器输入http://localhost:4000

同时还有对应的简化命令，好像后面每个单词首字

hexo n == hexo new

hexo g == hexo generate

hexo s == hexo server

hexo d == hexo deploy

hexo n == hexo new

部署Github

Hexo(3.2.0)文件配置

_config.yml repo属性

type: git

repository: git@github.com:15527918732/15527918732.github.io.git

branch: master

Github配置:

Github项目命名:账号+.github.io

GitHub项目配置:settings-GitHub Pages项目发布出去

3.我遇到的坑

  • 服务允许了，但是打开乱码_config.yml配置文件编码不是UTF-8,使用记事本打开另为UTF-8就好了
  • 修改配置文件的时候,不要乱删空格和空行.
  • 部署的时候,老是没有权限：先检查本地有没有密钥文件（~/.ssh）目录，没有的化根据网上配置，生成自己的密钥，同时把公钥复制下来，放github项目 settting-Deploy keys添加进去即可。
  • hexo部署问题：Error: spawn git ENOENT，hexo部署的时候应该也是一系列git命令的合集，需要git-server的支持。添加环境变量：C:\Program Files (x86)\Git\bin;C:\Program Files (x86)\Git\libexec\git-core，这样就解决了问题了。
  • 部署的时候老是报ERROR Deployer not found: git,执行一下npm install hexo-deployer-git –save这个命令
  • 网上下载.deploy_git影响了我的发布，它竟然把源代码提交上去啦，把他删除即可，git init重新生成就可以把自己的生成静态文件提交上去啦
